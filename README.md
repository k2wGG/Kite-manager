# Kite-manager — Скрипт для автоматики @Nod3r

- Автоматически обрабатывать транзакции и взаимодействовать с AI-системами.
- Управлять списком кошельков для обработки (файл `wallets.txt`).
- Работать через список прокси (файл `proxies.txt`) с автоматической ротацией.
- Редактировать и сохранять список вопросов для каждого бота (файл `endpoints.json`).
- Получать статистику и отчёты о взаимодействиях в режиме реального времени.

## Особенности

- **Интерактивное меню:** Простое меню для управления кошельками, редактирования вопросов и запуска автоматизации.
- **Конфигурация через файлы:**  
  - `endpoints.json` — хранит данные по конечным точкам, ID агентов и вопросы для каждого бота. Все изменения сохраняются и будут доступны при повторном запуске.  
  - `wallets.txt` — список кошельков (по одному на строке).  
  - `proxies.txt` — список прокси-серверов (один на строке, в формате `protocol://username:password@host:port` или `protocol:host:port`).
- **Подключение через прокси:** Скрипт автоматически использует прокси из списка и переключается на следующий в случае ошибки.
- **Московский часовой пояс:** Время выводится с учетом московского часового пояса.

## Установка

1. **Клонируйте репозиторий:**

   ```bash
   git clone https://github.com/k2wGG/Kite-manager.git
   cd Kite-manager
   ```

2. **Установите зависимости:**

   Убедитесь, что у вас установлен Node.js (рекомендуется версия 14 или выше).

   ```bash
   npm install
   ```

   Зависимости включают:
   - `node-fetch`
   - `chalk`
   - `readline`
   - `socks-proxy-agent`
   - `https-proxy-agent`
  
3. **Запуск скрипта:**

   ```bash
   npm start
   ```

## Конфигурация

### Файл `endpoints.json`

Этот файл содержит настройки для каждой конечной точки AI-систем, включая:
- `agent_id` — идентификатор агента.
- `name` — название системы.
- `questions` — массив вопросов для отправки.

Если файла нет, скрипт создаст его с дефолтными данными.

### Файл `wallets.txt`

Содержит список кошельков, на которые будет направлена автоматическая обработка. Каждый кошелёк указывается на новой строке.

### Файл `proxies.txt`

Содержит список прокси-серверов. Если этот файл отсутствует, скрипт работает в режиме прямого подключения.

## Использование

Запустите скрипт командой:

```bash
npm start
```

При запуске откроется главное меню с опциями:

1. **Управление кошельками (wallets.txt):**  
   Позволяет создавать или дополнять список кошельков.

2. **Редактировать список вопросов для ботов (endpoints.json):**  
   Изменяйте вопросы для каждого из ботов. Изменения сохраняются в файле `endpoints.json` и будут доступны при следующих запусках.

3. **Запустить автоматизацию:**  
   Скрипт начнёт автоматическую обработку транзакций, отправку запросов к AI-системам и вывод статистики.

0. **Выход:**  
   Завершает работу скрипта.


## Примеры вопросов для ботов

В файле `endpoints.json` можно задать по 30 вопросов для каждого бота. Примерные вопросы можно найти в описании проекта или редактировать через меню.

## Логотип

При запуске скрипта выводится логотип, оформленный с использованием ASCII-арта. Если логотип отображается «криво», убедитесь, что вы запускаете скрипт в терминале с моноширинным шрифтом (например, в Integrated Terminal VSCode или стандартном системном терминале).

## Примечания

- **Прокси:** Скрипт автоматически использует первый прокси из списка и переключается на следующий в случае возникновения ошибки.
- **Статистика:** Все взаимодействия и накопленные очки выводятся в консоль после каждого запроса.
- **Сохранение данных:** Изменения в списках вопросов и кошельков сохраняются между запусками скрипта.
